1. Critical Splunk Enterprise Flaw (CVE-2026-20253)

Splunk가 PostgreSQL sidecar 서비스 엔드포인트의 인증 부재로 인한 치명적 취약점(CVE-2026-20253, CVSS 9.8)을 긴급 패치했습니다. watchTowr Labs가 인증 없는 원격 코드 실행이 가능한 상세 공격 체인을 공개했습니다.

SplunkCVE-2026-20253RCEPostgreSQL
  • CVSS 9.8 — 최고 수준 위험도: Splunk Enterprise의 PostgreSQL sidecar 서비스 엔드포인트에 인증이 없어 원격 공격자가 인증 없이 파일 생성/변조 및 원격 코드 실행(RCE)이 가능한 취약점 CVE-2026-20253이 발견됨.
  • watchTowr Labs PoC 공개: 보안 연구팀이 /v1/postgres/recovery/backup 및 /restore 엔드포인트를 통해 공격자 데이터베이스를 Splunk 파일시스템에 덤프하고, 악성 Python 스크립트를 덮어써 RCE로 이어지는 공격 체인을 상세히 공개함.
  • 영향 및 패치: Splunk Enterprise 10.0.0-10.0.6 → 10.0.7, 10.2.0-10.2.3 → 10.2.4에서 수정. Splunk Cloud는 영향 없음. 아직 야생에서 악용되지 않았으나 PoC 공개로 공격 시도가 증가할 것으로 예상됨.
  • 공격 메커니즘: 공격자 DB 연결 → /backup으로 파일 덤프 → /restore로 악성 SQL 복원 → lo_export 함수로 파일 쓰기 → Python 스크립트 덮어쓰기를 통한 RCE

2. 미국, Anthropic에 Fable 5·Mythos 5 외국인 접근 차단 명령

미국 정부가 국가 안보를 이유로 Anthropic의 최첨단 AI 모델(Claude Fable 5, Mythos 5)에 대한 외국인 접근 차단을 명령했습니다. Anthropic은 '좁은 범위의 탈옥'을 이유로 한 이번 조치가 과도하다고 반박하며 법적 대응을 시사했습니다.

AnthropicFable 5Mythos 5AI Export ControlNational Security
  • 국가 안보 차원의 AI 수출 통제: 미국 정부가 Anthropic의 최첨단 AI 모델 Claude Fable 5와 Mythos 5에 대해 외국인의 접근을 전면 차단하라는 명령을 내림. Anthropic은 '오해가 있다'며 조속한 해결을 위해 노력 중.
  • Jailbreak 우려: 정부는 Fable 5를 탈옥(jailbreak)할 수 있는 방법이 발견되었다고 주장. Anthropic은 해당 기법이 '좁은 범위의 비범용적 탈옥'이며, 해당 취약점은 다른 공개 모델(GPT-5.5 포함)에서도 동일하게 발견된다고 반박.
  • Mythos 5의 사이버보안 능력: Mythos 계열 모델은 '세계 최강의 사이버보안 능력'을 보유하며, 새로 공개된 취약점을 몇 시간 내에 익스플로잇으로 전환 가능. 이는 기존 '월 단위 패치 주기' 모델을 무력화할 수 있음.
  • Anthropic의 반응: 회사는 투명하고 공정한 절차를 요구하며, 좁은 범위의 잠재적 탈옥이 널리 배포된 상용 모델을 리콜하는 이유가 되어서는 안 된다고 주장. DoD가 이미 Anthropic을 '공급망 위험'으로 지정한 바 있음.

3. ShinyHunters, Oracle PeopleSoft 제로데이로 100+ 기관 침해

ShinyHunters 갱단이 Oracle PeopleSoft 제로데이(CVE-2026-35273, CVSS 9.8)를 악용해 100개 이상 기관을 침해, 특히 대학들이 집중 표적이 되었습니다. University of Nottingham에서 약 455,000개의 개인정보가 유출되었습니다.

ShinyHuntersOracle PeopleSoftCVE-2026-35273Zero-DayUniversity Breach
  • Oracle PeopleSoft 제로데이(CVE-2026-35273, CVSS 9.8): ShinyHunters 갱단이 Oracle PeopleSoft Enterprise PeopleTools의 인증 없는 원격 코드 실행 취약점을 제로데이로 악용. Environment Management Hub(PSEMHUB) 컴포넌트가 취약점의 진입점.
  • 대학 68% 피해 집중: Mandiant가 확인한 100개 이상의 기관 중 68%가 고등교육 기관. University of Nottingham이 최초 확인 피해자로, 455,000개의 고유 이메일 주소와 개인정보가 유출됨.
  • 공격 세부 사항: 공격자들은 MeshCentral 원격 관리 에이전트를 Microsoft Azure 바이너리로 위장, Python SimpleHTTP 서버를 통해 C2 통신. SSH 스프레이 공격으로 내부 확산 후 데이터를 zstd 압축 후 유출.
  • 대응 및 탐지 지표: Oracle 권고: PSEMHUB 서비스 비활성화 또는 /PSEMHUB/* 및 /PSIGW/HttpListeningConnector 차단. WAF만으로는 우회 가능. WebLogic POST 로그, 의심스러운 .jsp 파일, XMLDecoder 영속성 메커니즘(outbound SMB 445) 확인 필요.
