논문 정보

제목: Who Pays the Price? Stakeholder-Centric Prompt Injection Benchmarking for Real-world Web Agents
저자: Zihao Wang, Yiming Li, Yutong Wu, Zheyu Liu, Kangjie Chen, Fok Kar Wai, Pin-Yu Chen, Vrizlynn L. L. Thing, Bo Li, Dacheng Tao, Tianwei Zhang
분야: AI Security / LLM Web Agent 보안

핵심 분석:

  • 문제 제기: LLM 기반 웹 에이전트가 실제 환경에서 신뢰할 수 없는 웹 콘텐츠 위에서 작동하며, 프롬프트 인젝션(악의적 지시를 포함한 콘텐츠가 에이전트 행동을 조작)에 취약하다. 기존 보안 벤치마크는 공격자 관점(attack-centric)에 치우쳐 있어 다양한 이해관계자(서비스 제공자, 에이전트 개발자, 사용자)의 위험을 종합적으로 평가하지 못한다.
  • 연구 제안: 8가지 실세계 웹 에이전트 시나리오에 걸쳐 이해관계자 중심의 프롬프트 인젝션 벤치마크를 제안한다. 공격자, 서비스 제공자, 에이전트 개발자, 최종 사용자 등 4개 이해관계자 그룹의 관점에서 각 공격 유형(직접/간접 인젝션, 데이터 도난, 악의적 행동 유발)의 영향을 체계적으로 정량화한다.
  • 작동 원리: 8개의 실제 웹 에이전트 태스크(쇼핑, 예약, 정보 검색, 파일 관리 등)를 설계하고, 각 태스크에 대해 다양한 프롬프트 인젝션 공격 벡터를 주입한 후, 각 이해관계자별 손실(cost)을 정량화하는 메트릭을 정의한다. 실험은 GPT-4, Claude 등 최신 LLM 기반 에이전트에서 수행되며, 공격 성공률 및 피해 규모를 다각도로 측정한다.

시사점 및 결론

이 연구는 LLM 웹 에이전트의 프롬프트 인젝션 취약점이 단순한 공격 성공률을 넘어, 누가 실제 피해를 입는지(stakeholder cost)에 대한 중요성을 부각한다. 기존 연구가 공격자의 관점에서 "얼마나 잘 공격할 수 있는가"에 집중했다면, 이 논문은 서비스 제공자의 평판 손상, 에이전트 개발자의 법적 책임, 최종 사용자의 금전적/프라이버시 피해 등 실질적 영향을 종합적으로 평가하는 프레임워크를 제시한다. 특히 웹 에이전트가 실제 서비스(전자상거래, 클라우드 스토리지 등)와 상호작용하는 시나리오에서의 취약점을 체계적으로 분석하여, 향후 LLM 기반 에이전트 시스템을 개발할 때 고려해야 할 보안 설계 원칙을 도출한 점이 핵심 기여다.
