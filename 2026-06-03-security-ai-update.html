1. 미아스마(Miasma) 공급망 공격 — Red Hat npm 패키지, 자격 증명 탈취 웜으로 추가 감염 확인

Miasma 공급망 공격 캠페인이 Red Hat의 @redhat-cloud-services npm 패키지를 표적으로 삼아 개발자 머신의 자격 증명을 탈취하며 확산되고 있습니다. Mini Shai-Hulud 계열 전술을 계승한 이 공격은 CI/CD 파이프라인까지 위협합니다.

  • 공급망 감염: @redhat-cloud-services/vulnerabilities-client 등 여러 패키지에 난독화된 preinstall 후크가 삽입되어 설치 시점에 GitHub Actions 시크릿, npm 토큰, SSH 키, 클라우드 자격 증명을 수집.
  • 자기 증식(웜) 특성: 탈취한 인증 정보를 통해 다른 저장소와 패키지로 자동 확산되며, 서명된 커밋으로 위장하여 탐지를 우회.
  • 클라우드 표적 확대: 이번 변종에서는 GCP 및 Azure 클라우드 아이덴티티 수집기가 추가되어 공격자가 인프라 자체에 대한 접근 권한을 탈취하려는 의도를 시사.
  • 지역 우회: 악성 코드는 러시아어 시스템에서 실행을 회피하여 특정 지역을 표적으로 하는 정교한 작전임을 암시.
  • 대응 방안: Red Hat 사용자는 즉시 영향을 받은 패키지를 업데이트하고 모든 API 토큰과 SSH 키를 로테이션해야 하며, CI/CD 파이프라인의 접근 권한을 최소화하는 것이 중요.

2. SeClaw: 자율적 LLM 에이전트 보안 평가를 위한 명세 기반 태스크 합성 프레임워크

최신 arXiv 연구(2606.02302)에서 발표된 SeClaw 프레임워크는 자율적 LLM 에이전트의 보안 취약점을 평가하기 위해 명세 기반 보안 태스크 합성과 실행 기반 평가를 결합한 혁신적인 접근법을 제시합니다.

  • 확장 가능한 평가: 기존 수동 태스크 중심 벤치마크의 한계를 극복하고, 자동으로 확장 가능하고 제어 가능한 보안 시나리오 구성을 가능하게 함.
  • 4대 위험 범주: 리소스 오용, 사용자 태스크 악용, 환경 취약점, 에이전트 내재 행동 위험 등 네 가지 위험 범주를 포괄적으로 다룸.
  • 실행 과정 기반 평가: 최종 응답만 평가하는 기존 방식에서 벗어나 에이전트의 전체 실행 과정(trajectory)을 분석하여 불안전한 행동의 전체 맥락을 진단.
  • 표준화된 테스트베드: Docker 기반 테스트 환경을 제공하여 다양한 LLM 에이전트의 보안 실패를 측정, 진단, 비교할 수 있는 실용적 기반을 마련.
  • 의의: 자율적 AI 에이전트가 실세계 작업을 수행하는 시대로 진입함에 따라, 이러한 에이전트의 보안성을 체계적으로 평가할 수 있는 프레임워크의 필요성이 크게 대두됨.

출처: arXiv:2606.02302

3. CERT-In, AI 기반 공격 대응 위해 인터넷 노출 취약점 12시간 내 패치 권고

인도 CERT-In이 AI 기반 사이버 위협의 급증에 대응하여 인터넷 노출 시스템의 취약점에 대해 12시간 내 패치 적용을 권고하는 새로운 가이드라인을 발표했습니다. 이는 공격자들이 AI와 LLM을 활용하여 공격 준비 시간을 극도로 단축하고 있기 때문입니다.

  • AI 가속 공격: 공격자들이 AI와 LLM을 취약점 발견, 악용 코드 분석, 피싱 콘텐츠 생성, 악성코드 제작에 활용함에 따라 공격 체인이 대폭 단축됨.
  • AI 시스템 자체 위협: AI 시스템은 프롬프트 인젝션, 데이터 유출, 젤브레이킹, 모델 조작, 훈련 데이터 오염 등 고유한 위협에 노출되어 있어 이중 대응 필요.
  • 위험 기반 패치 관리: 중요한 외부 노출 취약점은 12시간, 내부 시스템의 알려진 악용 취약점은 24시간 이내 조치를 권고하며 우선순위 기반 접근법 제시.
  • Zero Trust 강조: Zero Trust 아키텍처, 심층 방어 전략, Secure-by-Design 원칙을 채택하고 SBOM을 통한 소프트웨어 공급망 리스크 관리의 중요성 강조.
  • 글로벌 영향: 이는 정부 차원에서 AI 시대의 취약점 대응 속도를 공식적으로 규정한 사례로, 다른 국가들의 유사 정책 수립에 영향을 미칠 전망.
